# GitHub Setup Instructions

Follow these steps to push your repositories to GitHub:

## Step 1: Create GitHub Repositories

Go to [GitHub](https://github.com/new) and create three repositories:

1. **mentions** - Main repository (public or private)
2. **mentions-backend** - Backend service (public or private)
3. **mentions-frontend** - Frontend application (public or private)

> **Tip**: Don't initialize with README, .gitignore, or license - we already have these!

## Step 2: Add Remote URLs

Replace `YOUR_USERNAME` with your GitHub username:

```bash
# Main repository
cd /Users/amelton/mentions
git remote add origin https://github.com/YOUR_USERNAME/mentions.git
git branch -M main
git push -u origin main

# Backend repository
cd mentions_backend
git remote add origin https://github.com/YOUR_USERNAME/mentions-backend.git
git branch -M main
git push -u origin main

# Frontend repository
cd mentions_frontend
git remote add origin https://github.com/YOUR_USERNAME/mentions-frontend.git
git branch -M main
git push -u origin main
```

## Step 3: Update Submodule URLs

After pushing backend and frontend, update the main repository's submodule configuration:

```bash
cd /Users/amelton/mentions

# Edit .gitmodules file
# Change:
#   url = ./mentions_backend
#   url = ./mentions_frontend
# To:
#   url = https://github.com/YOUR_USERNAME/mentions-backend.git
#   url = https://github.com/YOUR_USERNAME/mentions-frontend.git
```

Or use these commands:

```bash
git config -f .gitmodules submodule.mentions_backend.url https://github.com/YOUR_USERNAME/mentions-backend.git
git config -f .gitmodules submodule.mentions_frontend.url https://github.com/YOUR_USERNAME/mentions-frontend.git

git add .gitmodules
git commit -m "Update submodule URLs to GitHub"
git push origin main
```

## Step 4: Sync Submodules

```bash
cd /Users/amelton/mentions
git submodule sync
git submodule update --init --recursive
```

## Step 5: Verify Setup

```bash
# Check remote URLs
cd /Users/amelton/mentions
git remote -v

cd mentions_backend
git remote -v

cd mentions_frontend
git remote -v

cd ..
# Check submodules
git submodule status
```

## Alternative: Using SSH URLs (Recommended for better security)

If you prefer SSH authentication:

```bash
# Main
git remote add origin git@github.com:YOUR_USERNAME/mentions.git

# Backend
cd mentions_backend
git remote add origin git@github.com:YOUR_USERNAME/mentions-backend.git

# Frontend
cd ../mentions_frontend
git remote add origin git@github.com:YOUR_USERNAME/mentions-frontend.git

# Update .gitmodules with SSH URLs
cd ..
git config -f .gitmodules submodule.mentions_backend.url git@github.com:YOUR_USERNAME/mentions-backend.git
git config -f .gitmodules submodule.mentions_frontend.url git@github.com:YOUR_USERNAME/mentions-frontend.git
```

## Troubleshooting

### If you need to change the remote URL later:

```bash
git remote set-url origin https://github.com/YOUR_USERNAME/new-repo-name.git
```

### If you get authentication errors:

1. Set up a [Personal Access Token](https://github.com/settings/tokens)
2. Use the token as your password when pushing
3. Or set up [SSH keys](https://docs.github.com/en/authentication/connecting-to-github-with-ssh)

### To verify everything is working:

```bash
git fetch origin
git status
```
